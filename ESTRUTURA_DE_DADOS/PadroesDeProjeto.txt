1. **O que são padrões de projeto?**

Padrões de projeto, também conhecidos como design patterns, são descrições de soluções prontas para problemas específicos que se repetem em projetos de software. Eles representam um modelo de resolução para problemas comuns, que pode ser aplicado em diversas situações¹. Eles surgiram na década de 1970, com o objetivo de auxiliar na solução de questões recorrentes¹. Os padrões de projeto representam as melhores práticas da programação orientada a objetos¹.

2. **Quais as vantagens em utilizar padrões de projeto?**

Os padrões de projeto podem proporcionar grandes benefícios, principalmente no que diz respeito à reusabilidade de código e, consequentemente, economia de tempo e dinheiro¹. Eles ajudam os desenvolvedores a resolverem problemas comuns que ocorrem em sistemas orientados a objetos através da reutilização de soluções que funcionaram no passado, tornando assim os sistemas mais flexíveis e reutilizáveis². Além disso, eles tornam o sistema mais fácil de entender e manter⁵.

3. **Características do Padrão de Projeto Strategy e quando é indicado aplicá-lo**

O Padrão Strategy é um dos padrões de projeto mais comuns usados em desenvolvimento de software orientado a objetos. Ele permite que você defina uma família de algoritmos, encapsule cada um deles e os torne intercambiáveis. Isso permite que o algoritmo seja alterado independentemente dos clientes que o usam⁸. 

O padrão Strategy é muito comum no código Java e pode ser usado em situações em que o seu código terá muitas variações de algoritmo e condições, gerando uma corrente de IFs⁷. Vale reforçar que não são todas as cadeias de IFs que precisam ser substituídas por um Strategy, apenas se as alterações forem frequentes a longo prazo⁷.
